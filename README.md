# Meal Planner

Статическое веб-приложение для планирования меню, расчёта списка ингредиентов и покупок.

## Возможности

- каталог из 30 рецептов с карточками, ингредиентами и пошаговым приготовлением;
- недельный план на 7 дней с четырьмя слотами питания;
- выбор одного или нескольких получателей для каждого дня;
- автоматическая агрегация ингредиентов по выбранным блюдам;
- группировка продуктов по категориям и ориентиры по срокам хранения;
- список покупок со статусами «куплено» и «есть дома»;
- локальное сохранение состояния в браузере;
- экспорт текущего состояния в JSON.

## Технологии

- Vanilla HTML, CSS и JavaScript;
- ES modules без фреймворка и внешних зависимостей;
- localStorage для локального состояния;
- готовность к подключению Supabase для авторизации и синхронизации.

## Структура проекта

| Файл | Назначение |
| --- | --- |
| `index.html` | Разметка приложения и четыре основных раздела |
| `styles.css` | Адаптивная визуальная система |
| `data.js` | Каталог рецептов и данные ингредиентов |
| `app.js` | Состояние, планирование, агрегация и список покупок |

## Локальный запуск

Проект использует ES modules, поэтому удобнее запускать его через небольшой локальный сервер:

```bash
python3 -m http.server 8080
```

После запуска открой в браузере: `http://localhost:8080`.

## Данные и приватность

Репозиторий не содержит персональных данных, токенов или секретных ключей. Состояние меню и списка покупок пока хранится только в браузере пользователя.

Не добавляй в репозиторий ключи Supabase, пароли и другие секреты. Для будущей серверной синхронизации конфигурация будет вынесена в переменные окружения.

## План развития

1. Проверить и расширить каталог рецептов и ингредиентов.
2. Добавить импорт рецептов и редактирование пользовательских блюд.
3. Подключить Supabase для синхронизации между устройствами.
4. Настроить публикацию статической версии и мобильный сценарий списка покупок.
5. Добавить резервное копирование и восстановление состояния.

## Статус

Текущая версия — функциональный MVP. Архитектура рассчитана на постепенное расширение каталога и подключение серверного хранения.
